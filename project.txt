# Projekt: Interaktive Web-Simulation zur Raumkrümmung durch Masse

## Ziel des Projekts

Erstelle eine browserbasierte, interaktive Simulation für den Physikunterricht in der EF am Gymnasium.

Die Simulation soll anschaulich zeigen, dass eine große Masse die Umgebung geometrisch „krümmt“ und dass sich dadurch die Bahnen anderer Körper sowie Lichtstrahlen verändern. Die Darstellung ist ausdrücklich **didaktisch modellhaft** und keine exakte numerische Simulation der Allgemeinen Relativitätstheorie.

Die Simulation soll besonders geeignet sein, um den Übergang von der klassischen Gravitation nach Newton zur Idee der Raumzeitkrümmung nach Einstein zu veranschaulichen.

---

## Zentrale didaktische Idee

Eine große feste Masse befindet sich im Zentrum der Simulation. Um diese Masse herum wird ein Gitter dargestellt, das die Krümmung des Raumes symbolisiert. Je größer die Zentralmasse eingestellt wird, desto stärker soll das Gitter verformt erscheinen.

Nutzerinnen und Nutzer können anschließend weitere Objekte hinzufügen:

1. **Testmasse**
   Ein kleiner Körper mit einstellbarer Anfangsposition und Anfangsgeschwindigkeit bewegt sich durch das gekrümmte Gebiet. Seine Bahn wird visualisiert.

2. **Lichtstrahl**
   Ein Lichtstrahl bewegt sich mit konstanter Geschwindigkeit durch das gekrümmte Gebiet. Seine Richtung wird durch die Krümmung beeinflusst. Dadurch soll qualitativ der Effekt der Lichtablenkung bzw. Gravitationslinsenwirkung sichtbar werden.

Die Simulation soll nicht den Eindruck erwecken, physikalisch exakt zu sein. Stattdessen soll sie ein klares, interaktives Modell für den Unterricht liefern.

## Darstellung der Simulation

### Canvas

Die Simulation soll auf einem großen Canvas dargestellt werden.

Der Canvas soll responsiv sein und sich an die Fenstergröße anpassen.

### Zentrale Masse

In der Mitte befindet sich eine feste Zentralmasse.

Darstellung:

* Kreis oder Kugel in der Mitte
* Größe visuell abhängig von der eingestellten Masse
* deutliche Markierung als „Zentralmasse“
* optional Beschriftung mit aktuellem Massenwert

Die Zentralmasse bleibt fest an ihrer Position.

### Gitter als Raumkrümmungsmodell

Um die zentrale Masse herum soll ein Gitternetz dargestellt werden.

Das Gitter soll abhängig von der Zentralmasse verformt werden.

Die Darstellung darf als 2,5D-Modell erfolgen:

* Das Gitter liegt eigentlich in einer Ebene.
* In der Nähe der Masse wird es optisch nach unten gezogen.
* Die Verformung wird perspektivisch oder pseudo-perspektivisch auf dem Canvas dargestellt.

Die Gitterverformung soll ungefähr durch eine Potentialfunktion beschrieben werden, zum Beispiel:

```text
z(r) = -warpStrength * M / sqrt(r² + softening²)
```

Dabei ist:

* `r` der Abstand zur Zentralmasse
* `M` die Zentralmasse
* `softening` ein kleiner Glättungswert, damit die Darstellung im Zentrum nicht unendlich wird
* `warpStrength` ein didaktischer Verstärkungsfaktor

Wichtig:

Die Gitterverformung soll deutlich sichtbar sein, darf aber nicht so extrem werden, dass die Simulation unlesbar wird.

### Trajektorien

Die Bahn von Testmassen und Lichtstrahlen soll als Spur dargestellt werden.

Die Spur soll dauerhaft sichtbar bleiben, bis sie gelöscht wird oder das Objekt zurückgesetzt wird.

Optional:

* ältere Spurbereiche transparenter darstellen
* maximale Spurlänge begrenzen
* Button „Spuren löschen“

## Physikalisches Modell

Die Simulation soll bewusst ein didaktisches Modell verwenden.

Bitte im Code und in der Benutzeroberfläche klar machen:

> Diese Simulation ist ein vereinfachtes Modell. Sie zeigt qualitativ, wie Massen die Geometrie der Umgebung verändern und dadurch Bahnen beeinflussen können. Sie ist keine exakte ART-Rechnung.

---

## Modell für Testmassen

Für Testmassen darf ein Newton-ähnliches Bewegungsmodell verwendet werden, weil es für Umlaufbahnen anschaulich und stabil ist.

Die Zentralmasse befindet sich bei:

```text
center = (0, 0)
```

Für eine Testmasse an Position `p` gilt:

```text
rVec = center - p
r = length(rVec)
a = G * M * rVec / (r² + softening²)^(3/2)
```

Dann Integration zum Beispiel mit semi-implizitem Euler-Verfahren:

```text
velocity += acceleration * dt
position += velocity * dt
```

Besser, wenn ohne großen Aufwand möglich:

* Velocity-Verlet
* Runge-Kutta 4

Für den Unterricht reicht aber ein stabiler semi-impliziter Euler-Integrator.

### Verhalten

Eine Testmasse soll je nach Anfangsgeschwindigkeit:

* in die Zentralmasse fallen
* an ihr vorbeifliegen
* eine gekrümmte Bahn beschreiben
* bei passenden Einstellungen eine Umlaufbahn zeigen

Wenn die Testmasse die Zentralmasse berührt oder einen Mindestabstand unterschreitet, soll sie als „kollidiert“ gelten und gestoppt werden.

---

## Modell für Lichtstrahlen

Lichtstrahlen sollen nicht einfach nach Newton wie Massen beschleunigt werden.

Stattdessen soll ein qualitatives Strahlenmodell verwendet werden.

Eine einfache Möglichkeit ist ein optisches Analogmodell mit einem effektiven Brechungsindex:

```text
n(r) = 1 + lensStrength * M / sqrt(r² + softening²)
```

Dabei ist:

* `n(r)` ein effektiver Brechungsindex
* `M` die Zentralmasse
* `lensStrength` ein didaktischer Verstärkungsfaktor
* `softening` ein Glättungswert

Die Richtung des Lichtstrahls soll sich in Richtung des Gradienten von `n(r)` ändern. Die Geschwindigkeit des Lichtstrahls bleibt betragsmäßig konstant.

Qualitativ:

* Licht bewegt sich immer mit konstanter Geschwindigkeit.
* Nur die Richtung wird durch die Krümmung verändert.
* Je näher der Strahl an der Zentralmasse vorbeigeht, desto stärker wird er abgelenkt.
* Je größer die Zentralmasse, desto stärker wird er abgelenkt.

Vereinfachter Rechenansatz:

```text
direction = normalize(direction)

gradN = gradientOfRefractiveIndex(position)

perpendicularComponent = gradN - direction * dot(gradN, direction)

direction += perpendicularComponent * dt * lightBendingFactor
direction = normalize(direction)

position += direction * c * dt
```

Wichtig:

Der Betrag der Lichtgeschwindigkeit soll konstant bleiben. Es soll keine Beschleunigung im Sinne einer Geschwindigkeitszunahme geben, sondern nur eine Richtungsänderung.

### Erwartetes Verhalten

Ein Lichtstrahl, der weit an der Zentralmasse vorbeiläuft, wird kaum abgelenkt.

Ein Lichtstrahl, der nahe an der Zentralmasse vorbeiläuft, wird sichtbar abgelenkt.

Bei sehr starker Zentralmasse und sehr nahem Vorbeiflug darf der Strahl stark gekrümmt werden. Er muss aber nicht physikalisch exakt um die Masse kreisen.

## Koordinatensystem

Intern soll ein Weltkoordinatensystem verwendet werden.

Beispiel:

```text
world x: -400 bis +400
world y: -300 bis +300
```

Der Canvas bildet diese Weltkoordinaten auf Bildschirmkoordinaten ab.

Die Zentralmasse liegt bei:

```text
(0, 0)
```

Der Renderer soll Funktionen enthalten wie:

```text
worldToScreen(position)
screenToWorld(position)
```

Für das gekrümmte Gitter kann eine zusätzliche Projektion verwendet werden:

```text
surfaceHeight(position)
projectSurfacePoint(x, y, z)
```

## Rendering

### Gewünschte visuelle Elemente

Die Simulation soll folgende Elemente darstellen:

1. Hintergrund
2. gekrümmtes Gitternetz
3. Zentralmasse
4. eingefügte Testmassen
5. eingefügte Lichtstrahlen
6. Bahnspuren
7. optional Vektorpfeile
8. kurze Legende

### Gitterdarstellung

Das Gitter soll aus Linien bestehen.

Für jede Gitterlinie werden viele Punkte berechnet. Für jeden Punkt:

1. Weltposition bestimmen
2. Abstand zur Zentralmasse berechnen
3. Oberflächenhöhe `z` berechnen
4. 2,5D-Projektion auf den Canvas berechnen
5. Linie durch die projizierten Punkte zeichnen

Dadurch soll die Umgebung der Masse sichtbar „eingedellt“ werden.

### Objekte auf der Oberfläche

Wenn Objekte und Spuren gezeichnet werden, sollen sie möglichst auf derselben gekrümmten Oberfläche liegen.

Dazu:

1. Weltposition des Objekts bestimmen
2. zugehörige Höhe `z = surfaceHeight(position)` berechnen
3. Punkt mit `projectSurfacePoint` auf den Canvas projizieren

## Bedienung per Maus

### Objekt hinzufügen

Die Bedienung soll möglichst intuitiv sein:

1. Nutzer wählt Objekttyp:

   * Testmasse
   * Lichtstrahl

2. Nutzer klickt auf den Canvas und hält die Maustaste gedrückt.

3. Beim Ziehen erscheint ein Pfeil:

   * Startpunkt = Klickposition
   * Pfeilrichtung = Bewegungsrichtung
   * Pfeillänge = Anfangsgeschwindigkeit

4. Beim Loslassen wird das Objekt erzeugt.

### Unterschiede

Bei Testmassen:

* Pfeillänge bestimmt die Anfangsgeschwindigkeit.
* Bahn wird durch das Testmassenmodell berechnet.

Bei Lichtstrahlen:

* Pfeillänge bestimmt nur die Richtung.
* Geschwindigkeit ist konstant.
* Bahn wird durch das Lichtmodell berechnet.

Beispiel:

```text
centralMass = 40

softening = 30
warpStrength = 180
simulationSpeed = 1

```

Diese Werte sind nicht fest vorgeschrieben. Sie sollen beim Entwickeln so angepasst werden, dass die Simulation stabil, gut sichtbar und didaktisch brauchbar ist.

Wichtig ist:

* Die Standardwerte sollen sofort einen sichtbaren Effekt zeigen.
* Lichtstrahlen sollen bei nahem Vorbeiflug sichtbar abgelenkt werden.
* Testmassen sollen je nach Startgeschwindigkeit verschiedene Bahnformen zeigen können.

## Stabilität und Grenzen

Die Simulation soll stabil laufen.

### Notwendige Schutzmaßnahmen

* Mindestabstand zur Zentralmasse verwenden
* Softening-Term verwenden, damit keine Singularität entsteht
* maximale Beschleunigung optional begrenzen
* sehr große Zeitschritte vermeiden
* Objekte entfernen oder stoppen, wenn sie weit außerhalb des sichtbaren Bereichs liegen
* Trail-Länge begrenzen, damit die Performance nicht leidet

## Pseudocode für den Simulationsloop

```text
function animationLoop(timestamp):
    berechne deltaTime

    wenn simulationRunning:
        für jedes Objekt:
            wenn Objekt aktiv:
                updateObject(object, deltaTime)
                neue Position zur Trail hinzufügen
                wenn Objekt kollidiert oder weit außerhalb:
                    Objekt deaktivieren

    renderer.clear()
    renderer.drawGrid()
    renderer.drawTrails()
    renderer.drawCentralMass()
    renderer.drawObjects()
    renderer.drawDragPreviewFallsAktiv()

    requestAnimationFrame(animationLoop)
```

## Wichtige didaktische Hinweise

Die Simulation soll keine falsche Exaktheit vortäuschen.

Bitte in README und UI deutlich formulieren:

* Das Gitter ist ein Modellbild.
* Die „Eindellung“ ist nicht wörtlich zu verstehen.
* Raumzeit ist vierdimensional; die Simulation zeigt nur eine vereinfachte zweidimensionale Analogie.

* Die Lichtablenkung wird qualitativ modelliert und didaktisch verstärkt.
* Ziel ist Veranschaulichung, nicht Forschungssimulation.
